# 退货简化：统一负数 + 谁退扣谁（ADR-019）

> 2026-07-22。在 ADR-018（C4 修复）之后进一步反思：精确匹配是否必要。

## 背景

原退货逻辑用 `src_order` 关联原销售做**精确匹配**（退货冲抵）+ 未匹配分流，做 3 件事：① 达成率用净额；② 退货提成归**原销售当天当班人**；③ 防同小票同条码多行重复冲减。

业务约束（行政手段）：**自己出的单自己退、全单退、金额对称**。
用户确认提成归属口径：**谁退扣谁**（退货提成归**退货当天当班人**，不归原销售员）。

→ 精确匹配唯一不可替代的作用（② 归原销售当班人）失效，可以删掉。

## 决策

退货统一按 amount 负数处理（= 原"未匹配退货"逻辑，推广到所有退货）：
- `commission = amount × rate`（amount 为负 → 提成负）
- 归**退货当天当班人** + 该人×店达成档
- 计入 `daily_sales`（退货天该店），达成率净额正确
- 台账标签统一「**退货**」（删「退货冲抵/退货未匹配」）

**不再用 src_order 关联原单号。**

## 等价性（用户场景下）

月聚合（人×店）下，简化与精确匹配结果等价：
- 达成率：销售净额 - 退货 = 净（daily_sales 含退货负）✓
- 提成：同人月聚合（谁退扣谁 + 行政保证退货员=原销售员）✓
- 重复冲减：全单退，无双行 ✓

## 影响

calculator：
- 删 C4 `excluded_index`（ADR-018）
- 删精确匹配（groups 的 returns 槽 + unmatched 分流）
- `group_net` 只算 sales
- `daily_sales` 含所有 returns（负）
- 步骤 6 删精确退货循环；所有 returns 走统一负数逻辑（tag=退货、退货天当班人、补 sales_record_id）

台账：「退货」统一标签，对账仍成立（退货 Σ 负 = 冲减）。

**撤销 ADR-018 的 C4 部分**（excluded_index）——简化后不存在精确/未匹配/C4 区分；漏洞1（sales_record_id）在统一逻辑里天然带（getattr）。

## 不变

- 步骤 1 分类（不计考核/赠送/非乳品/不计提成/excluded；销售/退货分流）不变
- 销售精确匹配（同 receipt+barcode 聚组成 net）不变——只为达成率净额，与退货无关
- 未匹配退货的「真·无原销售」问题消失（不再关联原单，退货独立算）

## 测试

- 销售 + 同人退货 → 月净提成 0
- 退货 tag=退货、commission 负、归退货天当班人
- 删原 C4 测试（test_gift_return_excluded_not_unmatched）——简化后赠送退货也走统一负数（commission 负，归退货天），不再是"归赠送剔除 0"。但赠送销售本身仍剔除（0）。
