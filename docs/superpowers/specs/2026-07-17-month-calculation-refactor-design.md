# 月度计算页面重构设计文档

> 创建日期：2026-07-17
> 状态：待审核

## 一、概述

重构月度计算页面，实现完整的计算流程管理、异常预检、排班确认和结果下钻功能。

### 核心变更

1. **月份卡片状态管理**：进行中/已计算两种状态，自动记录流程位置
2. **流水线步骤**：宽度自适应，非固定 Tab，步骤状态可视化
3. **异常预检**：6类异常清单，快速处理按钮，不离开当前页
4. **排班确认表格**：横向日期列，多人冲突红色高亮，支持拖拽调整
5. **结果下钻表格**：考勤天数/提成金额双入口展开，右侧抽屉明细
6. **薪酬制度集成**：自动关联生效版本，计算时锁定

新增字段：

```python
class Month(Base):

    current_step = Column(String, default="import")  # 当前步骤: import/targets/duty/results
```

### 2.2 步骤状态定义

```python
STEP_STATUS = {

    "duty": {"label": "当班确认", "order": 3},

```

### 2.3 异常记录模型

```python
class Anomaly(Base):
    __tablename__ = "anomalies"
    
    id = Column(Integer, primary_key=True)
    month = Column(String, nullable=False)  # YYYY-MM
    anomaly_type = Column(String, nullable=False)  # 1-6
    entity_type = Column(String)  # store/product/gift/refund
    entity_id = Column(String)  # 门店名/条码等
    description = Column(String)
    status = Column(String, default="pending")  # pending/ignored/resolved
    resolution = Column(String)  # 处理方式
    created_at = Column(DateTime, default=datetime.utcnow)
```

## 三、前端架构

### 3.1 页面结构

```
Months.tsx (月份列表)
    ↓ 点击月份卡片
MonthWorkspace.tsx (工作台)
    ├── StepIndicator.tsx (步骤指示器)
    ├── ImportStep.tsx (① 导入数据)
    ├── TargetsStep.tsx (② 配置目标)
    ├── DutyStep.tsx (③ 当班确认)
    └── ResultsStep.tsx (④ 计算&结果)
```

### 3.2 状态管理

**月份卡片状态：**
- `draft` / `computed`（现有）
- 新增 `current_step` 字段记录步骤位置

**点击行为：**
```typescript
// 进行中卡片 → 进入工作台，定位到 current_step
if (month.status === 'draft') {
  nav(`/months/${month.month}?step=${month.current_step}`);
}

// 已计算卡片 → 直接进入结果页
if (month.status === 'computed') {
  nav(`/months/${month.month}?step=results`);
}
```

### 3.3 步骤指示器

**宽度自适应布局：**
```tsx
<div style={{ display: 'flex', width: '100%' }}>
  {steps.map((step, i) => (
    <>
      <Step key={step.key} style={{ flex: 1 }}>{step.label}</Step>
      {i < steps.length - 1 && <Connector style={{ flex: 0.5 }} />}
    </>
  ))}
</div>
```

## 四、各节点详细设计

### 4.1 导入数据

**现有功能保持，增加：**
- 上传进度条
- 错误提示定位到具体行列
- 文件格式校验（列名校验）

### 4.2 配置目标

**新功能：**
- 底部汇总栏显示目标总额
- 直接创建新门店（弹窗），无需跳转
- 目标值范围校验（min 0, max 根据门店类型限制）

### 4.3 当班确认

**表格结构：**

```
| 门店 | 1号 | 2号 | 3号 | ... | 30号 |
|------|-----|-----|-----|-----|------|
| 福景店 | 张三 | 张三 | 李四 | ... | 王五 |
| 建设店 | 王五 | — | 王五 | ... | — |
```

**交互：**
- 正常单元格：显示当班人员名字
- 红色背景：同一店同一天多人当班，显示多个人员标签
- 悬浮人员名字：tooltip 显示交易笔数和营业额
- 点击红色单元格：下拉选择保留哪个人
- 拖拽人员到其他独占单元格：业绩关联过去
- "—"：无人当班

**数据来源：**
- 从销售流水中推断当班人员
- 存储到 Duty 表

### 4.4 薪酬制度自动关联

**规则：**
- 创建月份时自动关联当前生效的薪酬制度版本
- 显示：薪酬制度 v1（生效日期）
- 计算时锁定版本，确保历史可追溯

### 4.5 计算预检

**异常清单：**

| 类型 | 触发条件 | 处理按钮 |
|------|----------|----------|
| 异常1 | 交易流水门店不存在 | 创建门店（弹窗，自动代入名称） |
| 异常2 | 交易流水商品不存在 | 创建商品 |
| 异常3 | 门店无目标且未打不参与考核 | 填目标 / 打不参与考核标签 |
| 异常4 | 商品缺类别/成本且未打不计提成 | 编辑 / 打不计提成标签 |
| 异常5 | 赠送明细未匹配销售流水 | 忽略（再次计算不提示） |
| 异常6 | 退款明细未关联原订单 | 忽略 / 扣减营业额（负数累加） |

**异常列表展示：**
- 分类展示，每类一个折叠面板
- 每条异常一行，右侧快速处理按钮
- 不离开当前页，弹窗或行内编辑

**处理状态：**
- `pending` → 待处理
- `ignored` → 已忽略
- `resolved` → 已解决

### 5.1 第一层：人员汇总表

| 员工姓名 | 所在门店 | 门店类型 | 目标 | 日目标 | 考勤天数 | 实际目标 | 销售额 | 达标率 | 提成金额 | 汇总 |
|----------|----------|----------|------|--------|----------|----------|--------|--------|----------|------|
| 张三 | 福景店 | A | ¥84,000 | ¥2,800 | 5天 ▼ | ¥14,000 | ¥15,200 | 108% | ¥1,580 ▼ | ¥1,580 |
| 张三 | 建设店 | B | ¥50,000 | ¥1,667 | 3天 ▼ | ¥5,000 | ¥5,800 | 116% | ¥620 ▼ | ¥620 |

- 员工跨门店时姓名合并单元格
- 点击展开后行高自适应

### 5.2 考勤天数展开

点击「5天 ▼」→ 展开显示具体日期：

```
| 1号 ¥3,200 | 3号 ¥2,800 | 10号 ¥4,500 | 20号 ¥3,100 | 28号 ¥1,600 |
```

- 每个日期标签显示当日销售额
- 点击日期 → 右侧抽屉显示当天交易明细

### 5.3 提成金额展开

点击「¥1,580 ▼」→ 展开显示：

| 商品档位 | 销售额 | 提成比例 | 提成金额 |
|----------|--------|----------|----------|
| 常温高毛 | ¥6,500 | 12% | ¥780 |
| 常温低毛 | ¥3,200 | 7% | ¥224 |
| 低温高毛 | ¥4,000 | 13% | ¥520 |
| 低温低毛 | ¥1,200 | 9% | ¥108 |
| 特价 | ¥300 | 1% | ¥3 |

- 点击商品档位（蓝色）→ 右侧抽屉显示相关商品明细

### 5.4 双展开互斥

- 展开考勤天数时，提成金额保持折叠
- 展开提成金额时，考勤天数保持折叠
- 同一时间只能展开一个

### 5.5 明细标签

计算结果下钻的明细行需打标签：

- 🎁 **赠送**：赠送商品
- ↩️ **退款**：退款订单
- 🔄 **调整**：业务员调整到其他考勤日时

## 六、重新计算

**已计算卡片 → 右上角「重新计算」按钮：**

1. 点击确认对话框
2. 清除计算结果（Result 表数据）
3. 清除异常处理状态
4. 状态变为「进行中」
5. 回到导入数据步骤

## 七、后端 API 变更

### 新增端点

```
GET  /months/{month}/anomalies      # 获取异常列表
POST /months/{month}/anomalies/{id}/resolve  # 处理异常
POST /months/{month}/reset          # 重新计算（清除结果）
GET  /months/{month}/duty-grid      # 获取排班表格
PUT  /months/{month}/duty           # 确认当班（含拖拽调整）
GET  /months/{month}/transactions   # 交易明细（抽屉用）
```

### 修改端点

```
POST /months/{month}/compute        # 增加预检逻辑，异常未处理完拒绝计算
```
